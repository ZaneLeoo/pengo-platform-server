# AI 自动化业务规划

## 1. 定位与边界

自动化业务放在 `ruoyi-agent` 的 `com.ruoyi.agent.business.automation` 包中，
与 Dify 通讯、SSE 事件转发、普通只读查询工具分开。

自动化不是让模型直接写数据库，而是由模型收集意图，后端校验主数据和业务规则，
再由当前登录用户确认执行。统一链路如下：

```text
用户自然语言
  -> Dify Agent 调用“准备”工具（只读）
  -> 后端解析、补齐、校验主数据
  -> NEED_INPUT / AMBIGUOUS / INVALID / READY
  -> 用户确认标准化草稿
  -> 前端携带 RuoYi 登录态调用“执行”接口
  -> 后端创建草稿、记录创建人和审计信息
```

因此，Dify 的 `X-Agent-Tool-Key` 仅用于让 Dify 调用受限工具，不能替代 RuoYi 用户身份，
也不能作为最终写单据的授权凭据。

## 2. 已具备的数据基座

当前已存在并可用于自动化校验的基础档案：

- 供应商：编码、名称、币种、税率、状态；
- 物料：编码、名称、规格/型号、单位、状态；
- 仓库与库位：用于到货、质检、入库阶段；
- 采购订单、到货单、入库单、库存余额和库存流水。

采购订单只依赖供应商和物料。仓库、库位不应由采购建单阶段强制填写，
它们由后续“参照采购订单生成到货单”和“参照合格到货生成入库单”处理。

## 3. 第一项能力：AI 协助建立采购订单

### 3.1 准备工具：`preparePurchaseOrderDraft`

该工具是只读工具，允许传入不完整信息：

```json
{
  "supplierKeyword": "鸿发电子",
  "orderDate": "2026-07-12",
  "expectedDate": "2026-07-20",
  "lines": [
    { "materialKeyword": "PCB控制板", "quantity": 100, "unitPrice": 25.5 }
  ],
  "remark": "生产补料"
}
```

后端职责：

1. 按供应商编码或名称匹配启用供应商；无匹配、多个匹配时返回候选项；
2. 按物料编码或名称匹配启用物料；物料 ID、名称、规格、单位一律由后端带出；
3. 由供应商带出币种和默认税率，由后端重新计算每行金额、订单总量和订单总金额；
4. 校验数量大于零、单价不为负、日期格式正确、物料不可重复；
5. 返回结构化状态及标准化草稿，绝不创建采购订单。

返回约定示例：

```json
{
  "status": "READY",
  "message": "采购订单草稿已准备，请确认后创建。",
  "missingFields": [],
  "candidates": [],
  "draft": {
    "supplierCode": "SUP001",
    "supplierName": "深圳鸿发电子科技有限公司",
    "currency": "CNY",
    "orderDate": "2026-07-12",
    "lines": []
  }
}
```

### 3.2 执行接口：`createPurchaseOrderDraft`

执行接口不直接开放给 Dify。用户在前端看到 `READY` 草稿并点击“确认创建”后，
前端使用当前 RuoYi JWT 调用该接口。

后端必须再次解析供应商和物料、重新计算金额，然后：

- 自动生成订单编号、行号；
- 固定单据状态为 `DRAFT`，不允许 AI 直接审核；
- 写入当前登录用户的创建信息；
- 增加请求幂等键，避免重复点击或模型重试生成重复订单；
- 返回已创建订单编号和详情链接。

## 4. Dify 工具与提示词职责

OpenAPI 负责定义工具名称、字段、必填约束和返回结构，帮助 Agent 正确调用工具；
它不能自行实现多轮补问或写单据安全控制。

Agent 指令应明确：

1. 不得编造供应商、物料、单位、数量、价格或日期；
2. 先调用 `preparePurchaseOrderDraft`；
3. 对 `NEED_INPUT` 只追问缺失项，对 `AMBIGUOUS` 展示候选项让用户选择；
4. 对 `READY` 向用户展示汇总并请求确认；
5. 用户确认后，由前端执行创建；Agent 不调用审核、弃审或库存变更能力。

## 5. 后续能力队列

同一领域后续按相同边界扩展：

1. 根据采购订单生成到货单草稿；
2. 根据质检结果生成入库单草稿；
3. 库存预警分析与补货建议；
4. 生产、采购、库存数据的报告和文件草稿；
5. 审批待办提醒（只提醒，不替代审批）。

每项能力都先实现“准备/校验”工具，再实现用户确认后的执行接口。
